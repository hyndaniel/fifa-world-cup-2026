## Problem Statement

我(研究者/操盘者)想**实证理解"LLM 的独立推理能力,相对市场情绪与聪明钱判断,到底偏在哪、偏多少"**,而不是靠感觉断言"模型比市场准"或"市场永远对"。

具体三对问题(北极星,不变):

1. **v1 独立推理 vs 市场共识** —— 一个只从积分/出线形势/盘口/首发推理的脑,和去水后的市场共识,谁更准?
2. **v2(市场锚定 + 有据微调) vs 纯市场** —— 照抄市场再仅在有据处偏离,能不能系统性跑赢纯市场基线?
3. **v1 vs v2** —— 独立推理 vs 市场锚定,两种哲学谁的 Brier 更低、在哪类场型(常规/动机畸形)上分野?

痛点在于:要回答这三对问题,必须有**三方各自独立落库的预测 + 赛后实际结果 + 累积 Brier/分桶跑分卡**,且**预测时刻 v1 与 v2 绝不能互相看到对方**——否则"独立性"被污染,三方对照就失去意义。买票只是副产品,但它是让人**每天真金白银认真对待预测质量**的锚。

在有系统之前,预测散落在人脑和临时笔记里:没有独立落库、没有赛后回填、没有累积跑分卡、没有物理隔离 v1⊥v2、报告名互相冒充(`预测v2.md` 其实是三方跑分卡),"谁更准"从没被量过。

## Solution

一个 **LLM 多 Agent、按"描述 / 决策 / 调度"三层切分**的球赛分析系统,由**确定性 Workflow 脚本**焊死红线编排,产出落到一个**本地 SQLite 库 + 长青台账 md + HK 看板决策卡**,核心灵魂是 **Brier 三方对照 + 按场型分桶跑分卡**。

系统七个角色(`wc-<角色>-<版本>` 家族):

| 角色 | 层 | 职责一句话 | 现状 |
|---|---|---|---|
| `wc-score-v1` | 预测 | 独立推理比分/胜平负(先验+盘口+首发三层叠加),**不看 v2** | ✅ 已上线 |
| `wc-prob-v2` | 预测 | 市场锚定概率:照抄 baseline,仅有据偏离,**不看 v1** | ✅ 已上线 |
| `wc-odds` | 描述 | 取竞彩/欧盘/Poly 去水 → 共识/分歧/异动/陷阱结构,**不下价值判断** | ✅ 已上线(自 `odds-value-analyst` 裂出) |
| `wc-bet` | 决策 | 综合 v1/v2/共识 → value/+EV/最不亏腿/评方案/讲结算,**不进跑分卡** | ✅ 已上线(自 `odds-value-analyst` 裂出) |
| `wc-intel` | 情报 | deep-search 赛前情报 → 经 `save_intel` 喂三方(v2 只吃✅确证事实) | ✅ 已上线(deep-research skill + Workflow) |
| `wc-orchestrator` | 调度 | 判场 → fan-out 三脑 → 跑分卡 → join 决策卡 → 推看板;红线由脚本结构焊死 | 🟡 每日 routine 已成(跑今天 skill + fanout workflow);**事件驱动路由层未做** |
| `wc-review` | 复盘 | 赛后定性归因(仅假设、不下结论,裁判永远是定量跑分卡) | ⏳ 未落地(前段"机械对错标"✅,叙事 agent ❌,by-design 排最后) |

系统的**评测闭环**(灵魂)已完整上线:赛后 cron 自动抓终比分 → `record_result` → 重跑三方跑分卡 + 盖机械对错标;三方各自独立落库,v1⊥v2 由 Workflow 脚本结构(而非 agent 自觉)物理隔离。

## User Stories

### A. 评测闭环(北极星,系统灵魂)

1. 作为研究者,我要三方(v1/v2/市场)**各自独立**把每场预测落库,以便赛后能做干净的三方 Brier 对照。 ✅
2. 作为研究者,我要 **v2 在预测时刻绝对看不到 v1 的任何输出**(反之亦然),以便"独立推理 vs 市场锚定"的对照不被污染。 ✅(Workflow 结构焊死)
3. 作为研究者,我要赛后自动抓到每场真实比分并回填进库,以便跑分卡有 ground truth 可算。 ✅(`backfill_results` + cron)
4. 作为研究者,我要一份**累积、跨天连续**的三方跑分卡(Brier + 命中率),以便看谁更准、趋势如何。 ✅(`三方跑分卡.md`)
5. 作为研究者,我要跑分卡**按场型分桶**(常规 vs 动机畸形/末轮死局),以便判断"谁在哪类场上更准"而非只看总分。 ✅(`scorecard.bucket_of`)
6. 作为研究者,我要每场盖一个**纯机械对错标**(实际 had 对三方 argmax 的 ✅/❌),无 LLM、零讲故事,以便有一份不可争辩的事实底账。 ✅(`mech_tag` + `复盘对错标.md`)
7. 作为研究者,我要 `wc-bet` 和 `wc-review` 这类综合三方的下游产物**永不进跑分卡**,以便 Brier 只量三方各自独立的预测值。 ✅
8. 作为研究者,当某天预测漏落库(叙事 md 脱节),我要 routine 第一步能自动补齐,以便跑分卡不出现空洞。 ✅(跑今天 skill 第 1 步回填校验)

### B. 每日预测编排(wc-orchestrator)

9. 作为操盘者,我要一句「跑今天」就按 ET 当下时刻**自动判定该预测哪几场**,以便不用手工数赛程。 ✅
10. 作为操盘者,我要每场**并行**派 v1(比分)/ v2(概率)/ wc-odds(描述)三条独立 pass,以便一次跑完当日全部场次。 ✅(`wc-predict-fanout` workflow)
11. 作为研究者,我要 v2 的派单 prompt 是**入参的纯函数**(只含 market baseline + 中立事实卡),源里没有任何 v1 字段,以便红线是可审计的代码事实而非承诺。 ✅
12. 作为操盘者,我要三脑跑完后自动派 `wc-bet` 综合出 value/EV/最不亏腿,以便当日决策一步到位。 ✅
13. 作为操盘者,我要预测结果自动 join 成决策卡并推到 HK 看板,以便在网页上看当日全景。 ✅(decision_contract + `/api/ingest/predictions`)
14. 作为操盘者,我要在开球前 ~60min 的**首发微调窗**被自动提醒/触发一次,以便用最终首发名单修正预测。 ⏳(设计 §5#4,未落地)
15. 作为操盘者,当盘口出现**异动(线跳水/水位变化)或重大新闻**时,我要系统**自动判断该不该唤醒 deepsearch / 重跑某场**,以便不用整天盯盘。 ⏳(设计决策2"薄 LLM 路由层",未落地)

### C. 盘口描述(wc-odds)

16. 作为操盘者,我要读到竞彩/欧盘/Poly 三源去水后的**市场共识赛果**,以便知道"市场怎么看"。 ✅
17. 作为操盘者,我要看到**竞彩 vs 共识的分歧**,以便发现竞彩慷慨/苛刻的腿。 ✅
18. 作为操盘者,我要**盘口异动检测**(相对上一快照的线移动/水位变化),以便捕捉聪明钱动向。 ✅(`poller` 快照)
19. 作为操盘者,我要 `wc-odds` **只客观描述**共识/分歧/异动/陷阱结构与结算机制,**绝不**替我下价值判断,以便描述层和决策层不混。 ✅(边界写死,越界路由 wc-bet)
20. 作为操盘者,我要欧盘共识抓全 47 家(非首屏 30 家),以便中位数不失真。 ✅(翻页补全,PR#9)

### D. 下注决策(wc-bet)

21. 作为下注者,我要 `value = 竞彩欧赔 × Poly去水(p_true)` 与 **+EV/fair/−EV 分档**,以便量化每条腿划不划算。 ✅
22. 作为下注者,我要系统选出**最不亏的单关/串关腿**,以便"若要下,这条最不亏"。 ✅
23. 作为下注者,我要它**评我自己选的方案**(逐腿 value + 结算 + 诚实命中率/期望),以便知道我的串关值不值。 ✅
24. 作为下注者,我要它**讲清结算机制与陷阱规避**(如 +1.75 半赢半输、深盘低赔高门槛),用大白话 + 例子,以便看懂再决定。 ✅(台账大白话精简版,PR#79)
25. 作为下注者,我要一份**下注复盘台账**(结论先行 + 每押 1 元期望赚亏),以便回看历史决策质量。 ✅(`wc-bet__下注复盘.md`)
26. 作为下注者,我要系统**永不自动下注、不碰资金**,结论永远是"若要下,这条最不亏",以便安全边界清晰。 ✅
27. 作为下注者,当 Poly 数据陈旧时,我要在出价值结论前它被强制刷新,以便不被"伪造的最不亏黄档腿"误导。 ✅(记忆 [[poly-stale-false-yellow]])

### E. 情报层(wc-intel)

28. 作为研究者,我要一份 deep-search 赛前情报**经 `save_intel` 一次喂三方各取所需**,以便情报口统一、不重复采。 ✅
29. 作为研究者,我要 v2 **只吃✅确证事实**(带 factor_source)、真噪声零偏离,以便市场锚定不被小道消息带偏。 ✅(v2 intel-feed)
30. 作为研究者,我要每轮赛前情报**冻结成日期版快照**、事后不可改,以便保住"赛前那刻所知"的真盲测。 ✅(`reports/intel/YYYY-MM-DD__*.md`)
31. 作为研究者,我要 v1 能读情报**原文 md**(它三层叠加的正常输入),以便独立推理也吃到情报但不经 v2 的去噪通道。 ✅

### F. 命名与产出规范

32. 作为维护者,我要每份报告**用 agent-id 前缀**(`wc-score-v1__比分预测.md`),以便谁产的一眼可见、不再互相冒充。 ✅
33. 作为维护者,我要**台账类**(比分/复盘/跑分卡/对错标)长青单文件持续追加,以便累积指标不被拆日打碎。 ✅
34. 作为维护者,我要**快照类**(情报/盲测)日期版冻结,以便真盲测不被事后追写毁掉。 ✅
35. 作为维护者,我要非 agent 产出(跨模型实验、机器态 json)挪进 `_archive/`、`_state/`,以便 agent 语义空间干净。 ✅
36. 作为维护者,我要每个 agent.md 遵循**五段固定模板**(身份/边界/输入/输出/红线),以便边界一致可读。 ✅

### G. 复盘归因(wc-review,待落地)

37. 作为研究者,当三方跑分卡样本足够后,我要一个 `wc-review` agent 对"方向对/错 + 为什么"做**定性归因**,以便理解偏差成因而非只看数字。 ⏳
38. 作为研究者,我要 `wc-review` **只产出"假设 + 记录"、永远带"仅假设不下结论"帽子**,裁判永远是定量跑分卡,以便不在小样本上写"为什么错"的诱人散文。 ⏳(铁律护栏,设计 §4)

### H. 数据流转与看板

39. 作为操盘者,我要纯数据改动(台账/报告)能**直接 POST 到看板**、绕开 git/PR/部署,以便日常小改动快推。 ✅(数据快推 skill + `/api/ingest/tickets|reports`)
40. 作为操盘者,我要每日新拍的购彩票据**规范命名归档**并自动补录台账,以便购彩记录不散。 ✅(彩票入库 skill)
41. 作为操盘者,我要看板**决策卡靠队名 label join**(非 zucai_num),以便赔率面板不空。 ✅(记忆 [[odds-panel-join-key]])
42. 作为操盘者,我要看板密码只从 config.toml 现取、不 print / 不入库,以便 secret 不泄漏。 ✅

## Implementation Decisions

### 三层边界(决策1,不按数据源切,按"描述/决策/调度"切)
- **描述层 `wc-odds`** / **决策层 `wc-bet`** / **调度层 `wc-orchestrator`**;三脑(score/prob/intel)边界不动。
- `odds-value-analyst` 已裂成 `wc-odds` + `wc-bet`:共享读缓存代码(devig/consensus),各自 system prompt 收口(描述层不算 EV、决策层不取盘口)。✅

### 编排形态(决策2,混合)
- **骨架 = 确定性 Workflow 脚本**(`.claude/workflows/wc-predict-fanout.js`,JS,`parallel`/`phase`):控制流写死。✅
- **薄 LLM 层**只碰灰区("这条新闻算不算重大""异动算大算小"),输入严格白名单、不引跨脑污染。⏳(未落地)
- 拒绝纯 LLM 编排 agent(红线靠自觉迟早破)。

### 数据流红线(决策3,物理实现)——**核心不变量**
- 🔴 **v1 ⊥ v2 双向断开**,由 Workflow **三重结构**保证(非 agent 自觉):
  1. v1、v2 是同一 `parallel()` 里两个各自独立的 `agent()` 调用(各开上下文);并行派出 ⇒ 派 v2 时 v1 尚未返回。
  2. `v2Prompt(m)` 是入参 `m` 的**纯函数**,只读 `m.v2_baseline` + `m.v2_factcard`,源里无任何 v1 字段。
  3. v1 落 `v1_predictions` 表 + `wc-score-v1__比分预测.md`;v2 落 `v2_predictions` 表 + odds 缓存 + 事实卡——两套存储不相交。
- 🔴 **wc-orchestrator 只发"触发信号",绝不发"判断内容"喂脑**(路由器,非分析师)。
- 🟢 **wc-review 红线安全**:看全三方但全在赛后,预测时刻隔离未破。
- 🔴 **wc-bet / wc-review 不进 Brier**:跑分卡只量三方各自独立落库的预测值。

### 数据落点(schema / 存储)
- **DB**: `.cache/odds_cache.db`,四表 —— `odds_cache`(盘口快照)、`v1_predictions`、`v2_predictions`、`match_results`(ground truth)。写入均为 keyed upsert(按 match_key),并行安全。
- **DB 契约(deep module 接口)**:`record_v1(cache, match_key, {h,d,a}, 比分串)`、`record_v2_prediction(...)`、`record_result(...)`——三方落库/回填的唯一入口,agent 与 cron 都经此。
- **台账 md**(长青追加):`reports/agents/wc-score-v1__比分预测.md`、`reports/agents/wc-bet__下注复盘.md`、`reports/scoring/三方跑分卡.md`、`reports/scoring/复盘对错标.md`。
- **快照 md**(日期版冻结):`reports/intel/YYYY-MM-DD__赛前情报-*.md`。
- **看板 API 契约**:决策卡 dict 由本地生成 → `POST /api/ingest/predictions` → `db.save_decisions` 原样存 → `GET /api/decisions` 透传 → `frontend/app.js` 渲染;台账/报告经 `/api/ingest/tickets`、`/api/ingest/reports` 绕 git 落盘。

### 深模块清单(已建,可独立测试的稳定接口)
- `backend/devig.py` —— 去水(vig removal → 隐含概率)。
- `backend/baseline.py` —— 市场基线(v2 唯一合法锚:had/hhad/ttg 三盘口字符串产物)。
- `backend/scoring.py` + `backend/scorecard.py` —— Brier + 校准分桶 + `bucket_of`(场型)+ `aggregate`;`v2_report.py` 渲染成三方跑分卡。
- `backend/mech_tag.py` —— 机械对错标(实际 had 对三方 argmax 的 ✅/❌/—)。
- `backend/results.py` —— match_results / record_result(回填闭环终点)。
- `backend/v1_log.py`、`backend/decision_contract.py`、`backend/scenarios.py`(剧本库)、`backend/intel.py`、`backend/polymarket.py`、`backend/sporttery.py`、`backend/teammap.py`、`backend/devig.py`、`backend/bet_stats.py`。
- `backend/web.py` + `backend/run.py` + `backend/poller.py` —— 看板 API / 运行时 / 盘口轮询。
- `tools/`:`refresh_all`(三源刷新)、`backfill_results`(回填)、`save_intel`(情报喂三方)、`v2_report`(跑分卡)、`sim_match_montecarlo`(市场锚定蒙卡)、`push_data`/`ingest_client`(数据快推)、`settle_tickets`(结算)、`backtest_baseline`(基线回测)。

## Testing Decisions

- **好测试的定义**:只测**外部行为**(去水后的概率、跑分卡聚合数字、机械标的 ✅/❌、回填后的 record、决策卡 join 结果、看板 API 的落盘/透传),不测实现细节;红线不变量用**结构测试**兜底(v2 派单结构上不含 v1 字段)。
- **已覆盖模块(现有 prior art,tests/ 下已有对应)**:`test_devig`、`test_baseline`、`test_scoring`、`test_scorecard`、`test_mech_tag`、`test_results`、`test_backfill_results`、`test_v1_log`、`test_v2_predict`、`test_v2_report`、`test_decision_contract`、`test_decisions_view`、`test_value`、`test_odds_consensus`、`test_odds_store`、`test_polymarket`、`test_poly_proxy`、`test_intel`、`test_news_enrich`、`test_scenarios`、`test_settle_tickets`、`test_bet_stats`、`test_push_data`、`test_ingest_odds`、`test_refresh_align`、`test_reports`、`test_web`、`test_sim_match`、`test_state`、`test_config`、`test_db`、`test_sporttery`、`test_poller`、`test_backtest_baseline`。
- **新增需测模块(随 #4 / #5)**:
  - 薄 LLM 路由层:测**去抖 + 阈值**的纯函数部分(给异动幅度/新闻信号 → 该不该触发),LLM 判定部分只测输入白名单裁剪(不含跨脑内容),prior art = `test_news_enrich`。
  - 首发窗触发器:测 KO−60min 判定的**时间纯函数**(ET 铁律,温哥华 PT=ET−3),prior art = `gen_report_times` / `test_state`。
  - `wc-review`:测**机械前段已覆盖**(`test_mech_tag`);叙事后段是 LLM 产物,只测其台账追加格式与"仅假设"帽子存在性,不测归因内容。
- **红线回归**:新增/保留一条结构测试断言 `v2Prompt(m)` 的输出**不含**任何 v1 字段/输出(设计决策3 的可审计证据)。

## Out of Scope

- **自动下注 / 碰资金**:系统永远只给"若要下,这条最不亏",绝不代下、绝不管钱。
- **wc-review 下结论**:归因永远是"喂分桶卡去统计验证的假设",不是判决;裁判永远是定量跑分卡。
- **HK 直调模型**:订阅仅本地 macOS、无 API key → 模型只能 Mac 跑,HK 后端不直调,网页触发须任务队列桥接。
- **情报快照合并**:日期版快照只改名、绝不并成一个(毁盲测)。
- **v1⊥v2 之外放开隔离**:任何"让 v2 参考 v1 提高准度"的优化都出局——那毁掉北极星。
- **重 lint / 大框架**:编码规范只钉真在用的(UTF-8、优先 stdlib、`type(中文scope):` commit),不堆工具链。
- **ADR 正式化**:本 PRD 锁定的决定后续提炼成 `docs/adr/`(设计 §6 待决),不在本 PRD 范围内一次做完。
