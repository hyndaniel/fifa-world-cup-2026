# 淘汰赛蒙卡比分网格 → wc-bet 工具化 · 设计

日期:2026-06-29
状态:待实现(spec)

## 背景与动机

`tools/sim_match_montecarlo.py` 是市场锚定的二元泊松 + Dixon-Coles 比分网格引擎(PR#26 合 main)。
它从去水后的市场 1X2(90') + 大小球主盘反解双方期望进球 λ,建完整比分网格,再附一层
加时(30')+点球的晋级模型。当前只能**手改 CONFIG 再跑**。

用户想把后面的淘汰赛都模拟一遍。讨论澄清了一个关键事实:

- **蒙卡是市场锚定的,跟 v2 同一个锚**(λ 由去水赔率反解)。所以它的 1X2 跟 v2/wc-odds
  在结构上几乎一致、不是独立信号——**不能当第 4 个预测脑、不能进 Brier 跑分卡**
  (这正是 `wc-predict-fanout.js` v1⊥v2 红线要保护的)。
- 蒙卡真正独家、别人产不出来的是三样:**比分网格、热力图/Top 比分、加时+点球晋级%**。
  这三样的天然消费者是 **wc-bet**(决策终点,本来就需要网格算让球/净胜球腿)。

结论:**把蒙卡做成 wc-bet 调用的工具,只输出它独家的网格/热力图/晋级,不当独立预测、不进跑分卡。**

## 范围

做:
1. `tools/sim_match_montecarlo.py` 加"参数输入"模式,可被编排化调用。
2. wc-bet 在判淘汰赛让球/净胜球/大小球/晋级腿时调用该工具,摆出热力图+Top比分+晋级%,
   并用网格概率算腿的真实命中率。
3. Brier 红线写死在工具 docstring + wc-bet.md。

不做(YAGNI):
- 不新建 sim agent;不进 fanout 的预测 fan-out 相;不进 Brier 跑分卡。
- 不改 `跑今天`/`wc-predict-fanout` 的红线编排(小组赛阶段无晋级,本轮不动)。
- 不做夺冠概率链路/多轮推演(后续另议)。

## 详细设计

### A. 工具改造 `tools/sim_match_montecarlo.py`

新增 `--input '<json>'`(也支持 `--input -` 从 stdin 读),JSON 形如:

```json
{
  "match": "R32-76 Brazil vs Japan",
  "home": "Brazil", "away": "Japan",
  "p_home": 0.555, "p_draw": 0.258, "p_away": 0.187,
  "ou_line": 2.5, "p_over": 0.46,
  "pen_home": 0.5
}
```

- `--input` 提供时覆盖 CONFIG;**不提供则照旧用 CONFIG**(保留"模拟今晚XX"手动用法,不破坏现状)。
- `pen_home` 选填,**默认 0.5**(点球大战经验上近抛硬币;且它只乘在"90'平且加时还平"
  这一小块上,杠杆极低,硬填偏向值是给精度造假)。
- 输出两份:① 现有漂亮报告照旧打 stdout(热力图 + Top10 比分 + 晋级,即用户喜欢的部分);
  ② **永远**附机器可读 JSON(grid + summary + advance)。新增 `--emit-json` 把这份 JSON
  打到 stdout 末尾(以可解析的分隔标记包裹),供 wc-bet 直接捕获;`--json <path>` 另存沿用。
- 字段缺失/和≉1 时给清晰报错(沿用现有 `p_home is None` 的报错风格)。

### B. wc-bet 消费(改 `.claude/agents/wc-bet.md`,已获用户授权)

在 wc-bet 的工作流里加一条**淘汰赛专属**支路:

- 触发:当前场次是淘汰赛(有晋级诉求/加时点球结算)。
- 步骤:拿 wc-odds 的去水 1X2 + 大小球线 → 构造 `--input` JSON → `python3 tools/sim_match_montecarlo.py
  --input '<json>' --emit-json` → 解析返回的 grid/summary/advance。
- 用途:
  - 把**热力图 + Top 比分 + 晋级%** 摆进决策卡 / `reports/agents/wc-bet__下注复盘.md`。
  - 用网格概率算**让球腿/净胜N球/大小球腿的真实命中**(复用 handicap-leg-probability 那套:
    串关同向腿不连乘、深盘高门槛=陷阱、全包=零边际价值)。
  - 晋级% 跟 Polymarket 晋级盘交叉验证。
- 诚实声明(写进 wc-bet.md):蒙卡 1X2 与市场同锚,**不是独立证人**;比分形状是模型假设
  (二元泊松独立性 + DC ρ),不是实测——铁桶/对攻同 1X2 会得同网格。

### C. Brier 红线(写死)

- 工具 docstring + wc-bet.md §7 各加一句:蒙卡输出只活在 wc-bet 决策内部,
  **永不变成 v1/v2 式预测行、永不进 Brier 跑分卡**。

## 数据流

```
wc-odds(去水 1X2 + 大小球线)
        │
        ▼
wc-bet ── 构造 --input JSON ──▶ sim_match_montecarlo.py --emit-json
        ◀── grid / summary / advance ──┘
        │
        ├─ 热力图 + Top比分 + 晋级%  → 决策卡 / 复盘台账(展示)
        └─ 网格概率                  → 让球/净胜球/晋级腿 真实命中 → value/EV 分档
                                                                  (永不进 Brier)
```

## 测试 / 验证

1. 工具:`--input` 跑一场,确认报告与手改 CONFIG 一致;`--emit-json` 输出可被 `json.loads` 解析;
   `pen_home` 缺省走 0.5;`p_home` 缺失报错退非 0。
2. 端到端:挑一场已开赛淘汰赛(如 R32),wc-bet 走支路产出热力图+晋级%,人工核对晋级% 与 Poly 晋级盘量级相符。
3. 回归:不带 `--input` 的旧调用行为不变(CONFIG 路径)。

## 风险 / 限制

- 比分形状是模型假设,非实测(已声明)。
- 只有去水 1X2、没有大小球线时,总进球维度欠定 → `ou_line`/`p_over` 为**必填**,缺则报错退非 0(不静默退默认,避免造假总进球分布)。
- wc-bet 并行写 `wc-bet__下注复盘.md` 的既有写竞争风险不在本轮范围(沿用现状)。
