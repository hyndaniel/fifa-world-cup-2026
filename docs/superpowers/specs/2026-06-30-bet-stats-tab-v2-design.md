## 3. 诚实锚点（v2 关键变化）

v1 的锚点是「实购票全亏 −292 / ROI −100% / 0 中」。**收进 LYZ 06-28 那张 4/6 复式（+584.43）后翻面**：

- **推荐腿战绩块不变**（7/14、green=0、hypo +6.51/+46.5%）——那是模型选腿跑分，与这些真钱票互不相干。

## 5. 聚合层 `backend/bet_stats.py` 变更

`build_summary` 的 `tickets` 段改为（recommendations 段不变）：

```jsonc
"tickets": {
  "count": N, "settled_count": …, "pending_count": …,
  "won": …,                        // 已结票中 pnl>0 的张数
  "settled_stake": …, "settled_pnl": …, "settled_roi": …,   // roi=settled_pnl/settled_stake, 分母0→0
  "pending_stake": …,
  "by_person": [                   // 按 settled_pnl 降序, 同分按 who
    { "who":"LYZ", "tickets":3, "settled":1, "pending":2, "won":1,
      "stake":320, "settled_stake":114, "settled_pnl":584.43, "settled_roi":5.127, "pending_stake":206 },
    …
  ],
  "rows": [ …全部票, 含 settled 字段… ]
}
```

- 待结票（`settled=false`）：计入 `count`/`pending_count`/`pending_stake`/该人 `pending*`，**不计入** `settled_pnl`/`settled_roi`/`won`。
- `by_person` 用 `who` 分组聚合，已结/待结分开累计。
- 纯函数、stdlib，可单测（喂构造 ledger 断言分组与待结隔离）。
