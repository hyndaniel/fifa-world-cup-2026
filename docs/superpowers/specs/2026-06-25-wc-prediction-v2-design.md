# 世界杯预测系统 v2 设计

**market-anchored 概率预测 + 避雷器 + Brier 审判台**

日期:2026-06-25 ｜ 状态:设计已确认,待写实现计划

---

## 1. 背景与动机

v1(`reports/小组赛比分预测.md`)的**真盲测**(赛前落盘、非后视镜补记)战绩:

- 方向(胜平负)命中:**11/22 ≈ 50%**(基本抛硬币,且剧烈波动:6.15 全平 0/4、6.16 全悬殊热门 4/4)
- 精确比分命中:**2/22 ≈ 9%**

核心诊断:

1. **对着有效市场,点预测跑不赢。** 竞彩 + Polymarket 聪明钱 + 欧盘共识已把信息榨干(价值分析全程无 green/真错价即铁证)。-EV(抽水 ~11.5%、比分/半全场更深)是**结构性的**。
2. **靶子选错**:猜精确比分,9% 命中已是该目标理论天花板(足球最可能比分本身概率才 ~10-13%)。
3. **三层栈方向反了**:从弱先验微调向市场,最好只能复制市场,加自身噪声反而略差 → 50% 方向。
4. **系统性盲区**:低估热门净胜(2-0 撞大胜);对"大热门被摆大巴逼平 / 死亡橡皮擦轮换 / 默契平"这类反复出现的剧本失明。

**v2 的目标**(诚实定位):

- **不是**"更准的水晶球",**不承诺**把 -EV 变 +EV。
- **是**:(a) 校准、诚实、**可被数据检验**的概率预测;(b) 把模型最有用的能力——**识别该躲的混沌场(避雷器)**——做成一等输出。

**明确非目标**:不预测精确比分当 KPI;不替用户决定下不下注;下注的 +EV / 最短腿 / 小注判断仍归 `odds-value-analyst`。

## 2. 已锁定的设计决策

| 决策 | 选择 |
|---|---|
| v1 去留 | **冻结为基线 + v2 并行跑**;历史报告永不改 |
| v2 输出 | **全盘口校准概率**:胜平负 / 让球 / 大小球 / BTTS / 比分分布 / 半全场 |
| 概率生成 | **市场锚定基线 + 有据才偏离**(每次偏离记理由+结果,可审计) |
| 避雷器 | **一等输出**:每场「靠谱度(稳/中/乱)」+ 每盘口「可下/别碰」+ 命名剧本库 |
| 范围 | **一次建全**,瞄准淘汰赛 + 未来大赛(接受赶不上眼前一两场) |
| 架构 | **代码钉两端(基线+打分)、LLM 做中间判断(偏离+避雷+剧本)、v1 冻结当尺子** |

术语全部用大白话,见 §10 名词速查。

## 3. 架构总览 + 数据流

四个组件 + 一条冻结对照线:

| # | 组件 | 谁干 | 职责 |
|---|---|---|---|
| 1 | 基线引擎 | 代码 | 多源去水融合 → 每场「市场基线概率表」+ 诚实标锚硬不硬 |
| 2 | v2 预测 agent | LLM | 读基线,默认照抄,有据才偏离;出靠谱度 + 剧本标签 + 可下/别碰 |
| 3 | 剧本库 | 结构化文件 | 命名套路 + 触发条件 + 历史命中(自我证伪) |
| 4 | v1 predictor | 冻结 | 方法不动,继续出 v1 预测,只为对照 |
| 5 | 打分引擎 | 代码 | 赛后吃结果,算 v1/v2/市场 三方 Brier + 校准 + 偏离审计 |

数据流(一场比赛生命周期):

```
odds缓存(竞彩/Poly/欧盘)
   │  [基线引擎·代码:去水→融合→置信分级]
   ▼
市场基线概率表(全盘口 + 置信标:三源硬锚 / 单源软锚)
   │  [v2 agent·LLM:读基线]
   ▼
v2预测表(基线±偏离+理由 / 靠谱度稳·中·乱 / 剧本标签 / 每盘口 可下·别碰)
   │
   │  [v1 predictor·冻结] ──→ v1预测表(并行,互不读)
   ▼
 ……比赛进行……  →  实际结果回填(多源核对)
   │  [打分引擎·代码]
   ▼
跑分卡:v1 / v2 / 市场 三方 Brier + 校准 + 偏离是否=alpha → 报告置顶
```

**关键不变量:**

- 基线永远先落库、带时间戳、可复算 → 才比得了偏离有没有用。
- v2 默认 = 市场基线;偏离是例外,必须带理由。
- v1 与 v2 互不读对方 → 才是干净对照。

## 4. 组件 1:基线引擎(代码,确定性)

**职责:把缓存里的多源赔率变成每场一张「市场基线概率表」,并诚实标注每个盘口的锚有多硬。**

### 输入
读 `.cache/odds_cache.db` 每场、每盘口、每源最新快照(竞彩 / Poly / 欧盘共识)。固定取**赛前官方快照**(建议开赛前 ~1h、首发出来后)为该场**钉死基线**,用于赛后打分;更早快照仅作线移参考。

### 去水(复用 `backend/value.py` 口径)
- 竞彩欧赔:隐含 = 1/赔率,按盘口所有选项归一(Σ=1)。
- Poly:`ml / Σml`(已有 `poly_devig`)。
- 欧盘共识:30+ 家中位数欧赔去水(`odds_consensus` 的 `devig_pct`)。

### 融合 → 基线
- 基线 = 各源去水概率**加权平均**;权重可配,默认偏向聪明钱(Poly)与锐盘(欧盘共识)、压低深水竞彩。
- 每源的数单独落库 → 以后可重调权重;也供「源分歧」判断。

### 覆盖现实 + 置信分级(诚实核心)
| 盘口 | 源覆盖 | 锚 |
|---|---|---|
| 胜平负 | 竞彩 + Poly + 欧盘 | 🟢硬锚 |
| 让球 | 竞彩 + Poly(cover) + 亚盘(部分) | 🟢/🟡 |
| 大小球 | 竞彩 + Poly(ou) | 🟡中 |
| 比分分布 | 竞彩单源 | 🔴软锚(抽水深、无交叉) |
| 半全场 | 竞彩单源 | 🔴软锚 |
| BTTS | 由比分分布派生(Σ双方都进球的比分) | 🔴软(继承比分置信) |

外加「源分歧」标:三源去水差太大(如竞彩比共识慷慨 > 阈值)→ 标出,本身是信号(价值或陷阱),喂给价值脑。

### 输出
每场一张基线表:每盘口每选项的 `各源去水% / 融合基线% / 置信标 / 源分歧`,落新表(键=场次+盘口+快照时间),并渲染进报告。软锚盘直接标「🔴单源软锚,默认归避雷」。

### 复用 vs 新增
- 复用:`value.py`、`odds_watch.py`、`poly_fetch_hk.py`、`odds_consensus.py`。
- 新增:`backend/baseline.py`(融合+置信分级+落库)、竞彩比分/半全场 parser、`--selftest`(去水/融合/归一离线自测)。

## 5. 组件 2:v2 预测 agent(LLM,判断层)

**职责:拿 §4 基线表,默认照抄,只在有过硬理由时改动某个数,并标该不该碰。**

1. **读什么**:基线表(每盘口市场概率 + 置信标)。
2. **默认 = 照抄市场。** 多数盘口不改,直接用基线(市场已很准,乱改变差——v1 教训)。
3. **「偏离」= 改动市场那个数,必须有"能写下来的具体理由"。**
   - 例:054 捷克vs墨西哥,基线墨西哥胜 51%;赛前确认墨西哥轮换 10 人 → v2 下调 44%,理由记一行"锁头名+大轮换,市场没完全反映"。
   - 无具体理由 → 不准动,照抄。
   - 每条偏离记:盘口 / 原值→改后值 / 理由 / 信心。
4. **靠谱度(稳/中/乱)+ 可下/别碰**:每场打可预测度;每盘口标可下还是别碰。软锚盘(比分/半全场)默认别碰;乱场(默契平/大轮换)整场别碰。
5. **剧本标签**:从剧本库(§6)给这场贴标签 + 带出该剧本历史命中率。
6. **输出**:v2 预测表 = 基线 + (少量)偏离及理由 + 靠谱度 + 剧本标签 + 每盘口可下/别碰。

**纪律**:偏离要稀、要有据;一个剧本只有历史命中够高才有资格驱动偏离,否则只是提示标签;赛后 §8 用 Brier 审偏离到底是 alpha 还是噪声。

## 6. 组件 3:剧本库(会自我证伪的套路清单)

**剧本 = 反复出现、会把结果带偏的固定套路,起名记下。** 每个剧本四字段:

| 字段 | 含义 |
|---|---|
| 名字 | 好记的名 |
| 触发条件 | 什么情况算这个套路 |
| 典型影响 | 把概率往哪带 |
| 历史命中 | 触发几次 / 应验几次 → 命中率 |

「历史命中」让剧本库**自我证伪**:真能预测的留下加权,老应验不了的(迷信)降权或删。

**初始剧本(小组赛):** 大热门被摆大巴逼平(平/0-0↑)、死亡橡皮擦轮换(冷门/平↑)、默契平(平↑进球↓)、生死战必有胜负(平↓)、强队刷净胜球(大球/让球↑)。

**淘汰赛另起一套**(v2 瞄准淘汰赛):没出线动机这套,换成淘汰赛保守闷战、加时疲劳、强队点球魔咒等,随淘汰赛积累。

**用法 + 成长:** v2 每场扫库,触发即贴标签 + 抬高靠谱度"乱"的依据;赛后回填该剧本应验与否,更新命中记录。

**存哪:** `reports/剧本库.md`(或 yaml),v2 读 + 赛后回填。

## 7. 组件 4:v1 并行对照(冻结当尺子)

- v1 predictor **冻结**:提示词、方法不碰,继续出 v1 式预测(三层栈:一个比分 + 胜平负倾向)。
- 每场各记三份、互不通气:① v1 ② v2 ③ 市场基线。v1 与 v2 不读对方。
- 老 v1 报告永不改;v2 与对照另起新账。
- 价值:不留 v1,v2 只能跟市场比,无法证明它比老方法强。

---

## 8. 组件 5:打分引擎(代码,赛后自动)

**Brier = 给概率预测打的准度分,越低越准。** 算法 `(给的概率 − 实际)²`(实际发生=1、没发生=0)。说得准又敢说→分最低;嘴硬说错→罚最重;一直 50% 和稀泥→恒 0.25。

赛后算三件:

1. **三方 Brier 对比(分盘口)**:v1 / v2 / 市场,谁低。
   - v2 < 市场 → 判断真能跑赢;v2 ≥ 市场 → 跑不赢,老实只用市场。
   - v2 < v1 → 新方法强于老方法。
2. **校准检查**:v2 说 60% 的场真有约 60% 发生吗(按概率分桶比实际频率)。
3. **偏离审计(最关键)**:§5 的偏离平均把 Brier 拉低还是拉高 → 回答"足球判断是 alpha 还是噪声";顺带回填 §6 剧本命中。

**输出**:跑分卡置顶报告——三方准度分、校准曲线、偏离是赚是亏。这是整套设计的**审判台**。

## 9. 落地 + 错误处理 + 测试

### 9.1 落地

- **代码**:复用 `value.py`/`odds_watch.py`/`poly_fetch_hk.py`/`odds_consensus.py`;新增 `backend/baseline.py`、`backend/scoring.py`、竞彩比分+半全场 parser。
- **数据**:`.cache/` 内新表——基线表、v2 预测表、v1 预测表、实际结果表、跑分表(键=场次+盘口+快照时间)。
- **报告**:老的不动(`小组赛比分预测.md` 冻结、`盘口下注复盘.md` 照旧);新 `reports/预测v2.md`(置顶跑分卡 + 每场明细)、`reports/剧本库.md`。
- **Agent**:v1 predictor 冻结;新建 **v2 预测 agent**;`odds-value-analyst` 改吃 v2 标"可下"的盘口算价值。一条编排把三份一起跑、一起记。

### 9.2 错误处理(原则:缺数据降置信 + 标注,绝不编)

| 情况 | 处理 |
|---|---|
| 某源缺 | 退化为现有源,降置信;只剩竞彩→🔴软锚 |
| 三源全缺该盘口 | 不出基线,标"无盘"、跳过 |
| Poly 过旧 | 走 `poly_fetch_hk` 刷;刷不动用旧的+标X小时旧,或从融合踢掉 |
| 比赛延期/腰斩 | 标记、排除出打分(不污染 Brier) |
| 首发未按时出 | 基线照取+标"首发未出",后续快照补 |

### 9.3 测试

- **数学自测**:去水/融合/归一 Σ=1、Brier 算对(已知输入→已知分),照 `odds_watch --selftest` 体例。
- **样例测试**:罐头赔率 → 核对基线表。
- **回测(最值)**:重放已踢完的 6.11–6.24,只用市场基线算 Brier——① 验证市场基线校准;② 立起「市场 Brier」基准线(v2 必须打败的靶子)。
- **试跑**:剩余小组赛/淘汰赛先 dry-run 跑通管线再正式上。

## 10. 名词速查(大白话)

| 词 | 大白话 | 例 |
|---|---|---|
| 去水 | 抹掉庄家抽成,还原成加起来100%的真实概率 | 韩国胜@1.44→去水后62% |
| 各源去水% | 每个来源各自去水后的概率 | 竞彩62/Poly61/欧盘58 |
| 融合基线% | 多源加权合成一个,作 v2 起点 | ≈61% |
| 置信标 / 锚 | 这个数多可信:多源=🟢硬,单源=🔴软 | 胜平负=硬,半全场=软 |
| 源分歧 | 各源彼此差多少;差大=信号 | 竞彩62 vs 欧盘58,小分歧 |
| 偏离 | 把市场的 X% 改成 Y%(须有理由) | 51%→44%,理由轮换 |
| 靠谱度(稳/中/乱) | 这场多好预测;乱=别碰 | 默契平=乱 |
| 剧本 | 反复带偏结果的固定套路 | 大热门被摆大巴逼平 |
| Brier | 概率预测的准度分,越低越准 | 说61%且应验,罚0.15 |
| BTTS | 双方都进球(是/否) | 2-1=是,2-0=否 |

## 11. 红线 / 非目标

- **不承诺 +EV**;复式 ≠ 正期望;-EV 是结构性的。v2 改善的是诚实、校准、避雷,不是盈利。
- 缺数据降置信、绝不编。
- v2 = 预测 + 避雷;**下注决策(+EV、最短腿、小注)仍由 `odds-value-analyst` 出**。
- **诚实兜底**:若 v2 长期跑不赢市场基线(Brier 不优于市场),明确认怂,回归"只用市场基线 + 避雷器",不硬撑。

---

## 12. 成功标准(怎么算这套有用)

1. **回测通过**:市场基线在 6.11–6.24 上校准合理,基准 Brier 立起。
2. **上线后约 20-30 场**:v2 Brier ≤ 市场 Brier(至少不更差);偏离审计显示偏离平均**不拉高** Brier。
3. **避雷器有效**:标"乱"的场,事后确实更难预测 / 更易爆冷(命中率显著低于标"稳"的场)。
4. **若 1-3 不达标**:按 §11 兜底认怂,退回市场基线 + 避雷器,本身也是诚实的成功(止损 v1 式自欺)。
